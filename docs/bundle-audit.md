# Аудит бандла (Next.js)

## Как посмотреть визуализацию

1. Установите зависимости (`npm install`).
2. Выполните **`npm run analyze`** — соберётся production-сборка с включённым `@next/bundle-analyzer` (webpack).
3. В конце сборки откроется интерактивный отчёт (treemap по чанкам). Статические файлы также лежат рядом с артефактами сборки (см. вывод в терминале).

Скриншот-пример treemap: [`bundle-analyzer-treemap.png`](bundle-analyzer-treemap.png) (иллюстрация; актуальную картину всегда смотрите после своего `npm run analyze`).

## Тяжёлые зависимости (типично)

- **Next.js + React** — ядро клиентского рантайма и фреймворка.
- **Redux Toolkit / react-redux** — стор и подписки.
- **Отдельные async-чанки** после разбиения:
  - окно чата (`ChatWindow`) подгружается с маршрута;
  - сайдбар и панель настроек — ленивые чанки;
  - **react-markdown**, **remark-gfm** и **Prism** подтягиваются с ленивого модуля `AssistantMarkdown` (не в «первом» экране без сообщений ассистента).

## Code splitting Markdown / Prism

Компонент **`AssistantMarkdown`** подключается в `Message.tsx` через `React.lazy` + `Suspense`, поэтому **react-markdown** и **Prism** не входят в тот же синхронный чанк, что и лёгкая оболочка сообщения.

## Profiler (React DevTools)

После мемоизации `ChatItem`, `useMemo` для отфильтрованного списка и стабильных `useCallback` у обработчиков в `ChatList` / `ChatWindow` имеет смысл записать сессию **Profiler** при:

- отправке сообщения в одном чате (остальные `ChatItem` не должны лишний раз коммититься);
- вводе в поиске (перерисовываются только отфильтрованные элементы и зависящий UI).
